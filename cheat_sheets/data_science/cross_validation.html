Cross validation

Information

In cross-validation, a dataset is first split into folds.

In each experiment run, all folds except one are used for train and the remaining fold is used for validation.

The number of folds is equivalent to the numbero of folds, i.e., each fold is used once for testing.

This means that the result of cross-validation is a score for each fold.

Typically the mean of the score is reported along with the standard descriptive statistics about the distribution of the scores.

Cross-validation generally prevents overfitting better than a single split into training and validation.

However, cross-validation is computationally more expensive.

There are different types of cross-validation. The main types are:

  • K-fold cross-validation split into n folds
  • Stratified K-fold Cross Validation The splitting of data into folds may be governed by criteria such as ensuring that each fold has the same proportion of observations
  • Leave One Out Cross Validation (LOOCV) each data instance is a fold. The number of runs corresponds to the number of folds.
Usage

from sklearn.model_selection import cross_val_score can be used for cross-validation.
