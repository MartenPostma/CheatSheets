the syntax to search is db.COLLECTION.find({})

Any key:value pairs you specify in {} limits the number of results

to match on id_, you have to provide it as ObjectId("KEY")

You can project which fiels will be shown in the output usng a second dictionary: db.COLLECTION.find({}, {})

For the projection dictionary, you indicate per key which one you want to see, e.g., {"name":1, "email":1}

You can a nested search using KEY_LEVEL1.KEY_LEVEL2.

If the value of key is a list of values, e.g, "email" : [ "lhmdebruin@hotmail.com", "leontien134@tiscalimail.nl" ]

you can still retrieve them using {"email" : "lhmdebruin@hotmail.com"}.

and & or: you can use $or, $and to provide a list of queries
