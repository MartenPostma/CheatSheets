The Event Loop

Information

Node.js runs on a single thread.

Typically, there is one event loop per browser tab.

The goal is to write code that does not block one event loop.

Blocking the event loop

Any JavaScript code that takes too long to return back control to the event loop will block the execution of any JavaScript code in the page.

Being blocked is the exception, which is why Javascript is based on callbacks, promises, and async/await.
