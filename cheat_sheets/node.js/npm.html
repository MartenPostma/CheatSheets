npm

Information

npm is the standard package manager for Node.js.

You can install a package from the command line using

npm install PACKAGE_NAME

You can update using

npm update

will check all packages for a newer version that satisfies your versioning constraints.

Package.json

If a project has a package.json, you can install all dependencies using

npm install

The packages will be stored in the node_modules folder.

Important flags

--save adds the package to the key dependencies (production) of package.json

--save-dev adds the package to the key devDependencies (development) of package.json

There are at least three ways to add a package to package.json

{

"dependencies": {

"express" : "4.17.1"

"express" : "~4.17.1"

"express" : "^4.17.1"

}

This is what it means:

  • "4.17.1": install exact version
  • "~4.17.1": will update you to all future patch versions, without incrementing the minor version
  • "^4.1.7.1": will update you to all future minor/patch versions, without incrementing the major version

You can also use package.json to run tasks

npm run task_name

example:

{

"scripts": {

"start": "node app.js"

}

You can run this task using npm run start
